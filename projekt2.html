Large Language Model Ollama

Ein Large Language Model (LLM) ist eine Art von künstlicher Intelligenz, die darauf ausgelegt ist, menschliche Sprache zu verstehen und zu erzeugen. Es wird mit großen Mengen an Texten trainiert, um zu lernen, wie Sprache funktioniert und wie auf bestimmte Eingaben sinnvoll reagiert werden kann. Diese Modelle erkennen Muster in den Daten und können so beispielsweise Texte verfassen, Fragen beantworten oder Dialoge führen.

Ollama ist ein Werkzeug, das es ermöglicht, ein Large Language Model lokal auf einem Computer auszuführen. Ollama bietet ein sogenanntes Command Line Interface, über das der Benutzer mit dem LLM interagieren kann.

Im Rahmen des Projekts wurden zwei Modelle von Ollama verwendet: zum einen das Tinyllama, eine kleinere Version des LLM, die deutlich mehr Zeit zur Berechnung von Ergebnissen benötigt, und Ollama3, ein größeres Modell, das über mehr Parameter verfügt, jedoch auch länger für die Generierung von Antworten braucht.

Das Projektkonzept bestand darin, eine Anwendung zu entwickeln, die in der Lage ist, eigenständig 2D-Bilder von Häusern zu konstruieren, um Menschen mit weniger Kreativität eine Inspiration für den Bau eines Hauses zu geben, das aus mehr als nur Wänden und einem Dach besteht. Um dies zu erreichen, wurden in Adobe Illustrator Häuser entworfen und als SVG-Code exportiert. SVG ist eine Auszeichnungssprache, die Bildinformationen in Vektoren, Kreisen und anderen einfachen Formen beschreibt. 

                    
                        <svg width="100" height="100">
                            <circle cx="50" cy="50" r="40" fill="red" />
                          </svg>

Dieser SVG-Code beschreibt einen roten Kreis in einem 100px x 100px großen Bereich mit einem Radius von r="40" und der x-, y-Position cx="50", cy="50".

Ollama funktioniert, indem Rollen zugewiesen werden, um die Art und den Stil der Antworten zu beeinflussen und zu konkretisieren. So kann beispielsweise die Rolle "narrator" für eine erzählerische Antwort oder "system" für eine präzise, den Anforderungen entsprechende Antwort festgelegt werden. Diese Anforderungen, oder Anweisungen, können in Form von "content" übergeben werden. Hier ein Beispiel aus dem Code:

role: `system` content: `You are an SVG generator. You only answer in SVG code. NO markdown. NO backticks. Only the svg xml. Black is only to appear in the same areas as #D9D9D9. The generated code will be used as real svg. This is the SVG-code for a HOUSE, everything between three quotes is the example code: ...`

Anschließend folgt ein konkretes SVG-Beispiel. Die Rolle "system" hat die Aufgabe, basierend auf dem bereitgestellten SVG-Code neuen, ähnlichen SVG-Code zu generieren, ohne dabei Markdown oder Backticks zu verwenden, damit der generierte Code direkt weiterverwendet werden kann.

Durch die Verwendung von Großbuchstaben (CAPS) wird festgelegt, dass bestimmte Anweisungen priorisiert werden. Wenn beispielsweise Markdown im Output erscheint, kann der Prompt so angepasst werden, dass mit "NO markdown" sichergestellt wird, dass kein Markdown generiert wird.

Die Rolle "assistant" unterstützt die Rolle "system" in diesem Projekt, indem sie weiteren SVG-Code bereitstellt, der dem "system" zur Generierung zusätzlicher Inhalte hilft. Eine wichtige Funktion ist die sogenannte "temperature". Diese bestimmt den kreativen Spielraum des LLM und legt fest, wie sehr sich das Modell an die Vorgaben halten soll. Bei einem niedrigen Wert weicht das LLM kaum von den Beispieldaten ab, während bei einem hohen Wert die generierten Ergebnisse größere Abweichungen aufweisen können.

Das LLM hat damit eine Aufgabe und gibt als Output SVG-code, welcher direkt in den Html-code gezeichnet wird. Für mich weicht der Output noch nicht weit genug von den Beispieldaten ab. Zudem benötigt das LLm viel Zeit, bis der Output generiert wird.

Das Projekt ist unter https://housecreator.netlify.app/ zu erreichen.
